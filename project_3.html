Prepared

In this project I was mostly working on User Interface. The goal is to design an app that lets users plan their trip more reliably. The end user that the app is designed for, is someone who has trouble planning out their trip. Namely knowing what to do during the trip is something the user struggles with. I chose 3 API's for the app to use in theory. The API's helped defining the functionalities of the app. One of the functionalities revolves around an API that searches for bars in the environment, so the user can find activities to during the trip. The functionality is designed in a way that the app can be used before, during and after the trip.

Learning points

During this project, I learned about API's, but didn't completely make use of them. It would be interesting to try once. I applied design patterns in designing the screenflow for functionalities like giving reviews. So I had to analyze the design patterns well to implement them in my own design. The app has to be used in different phases of a trip, which is an approach I've never elaborately taken. This made it so I took into consideration that users want to do different things in a variety of scenarios.
